Installation Instructions for Examples

Please refer to the following information for instructions on how to install and configure the PowerBuilder Internet Tools examples.

Installation Steps:

  1. Installing Sample Files
  2. Preparing the Server Application for Networking
  3. Preparing the Server Application for Database Access
  4. Preparing your Web Server for PB Plug-ins
  5. Running the Examples. (Steps 1-4 must be completed first)

Installing Sample Files

Before running any of these examples, you must copy the following files from the IT\Examples directory into a document directory on your HTTP (Web) server:
(i.e. If you have installed WebSite, this directory defaults to C:\Website\htdocs)

  • sample.htm
  • webpb.htm
  • winplug.htm
  • dwplug.htm
  • winplug.pbd
  • dwplug.psr
  • pb1.jpg

Preparing the Server Application For Networking

Setting up the application's TCP/IP port

To allow the PowerBuilder Web Examples Application server to run as a TCP/IP application, you must assign it a TCP/IP port. To do this, you must add the PowerBuilder application name to your network SERVICES file. The file is called SERVICES (with no extension) and usually resides in your Win95 directory. Windows NT users will find the SERVICES file in \WINNT\SYSTEM32\DRIVERS\ETC. The reference you need to include should appear as follows: 

        pb_net_examples 10098/tcp

In our example line for your SERVICES file, found in servicpb.txt, we specify TCP port 10098. If that port is listed in your SERVICES file as being occupied by another application, you must change the port to another number that is not already in use.

Setting up the Application's CGI Interface

To have the ability to execute these examples, the PowerBuilder Web Server Application must be running, and the PowerBuilder Common Gateway Interface (CGI) executable must be installed in a script/executable directory on your Web (HTTP) server.

If you have installed WebSite, we will be placing the PowerBuilder CGI executable in the Web server's /cgi-shl/ directory. If this is not your configuration, then you must manually place the PowerBuilder CGI executable, PBCGI60.EXE in a Web server script/executable directory. Once you have installed the CGI executable in that directory, you must update the links in the file webpb.htm to reflect this new directory as well as update the instance variable is_CGIURL in user object u_webpbexample that is part of the webpbsrv.pbl. We recommend rebuilding the .exe and .pbd's if you make this change by using the project object included in the .pbl.

For example, if I installed the file PBCGI60.EXE into an executable directory on my Web server called scripts/, then I could do a search and replace on cgi-shl/ and replace it with scripts/.

Once that change has been made, you should be able to load the sample files from your Web server and follow the links to the samples.

Preparing the Server Application for Database Access

Some of these examples rely on SQL Anywhere 5.x, the Powersoft Demo DB V6 and the Web.PB Class Library being installed in your system. The ODBC datasource profiles that will be used for these connections are "Powersoft Demo DB V6" and "Powersoft WebPB". Please ensure that these ODBC profiles still exist before running the server application.

NOTE: Turning off AutoStop Database in the required ODBC profiles will ensure that the database remains running throughout your viewing of the sample applications.

Preparing the Server Application for Execution

In order to run the server application, add a program shortcut to one of the folders on your workspace. The shortcut should identify the server application as the target, for example: "C:\Program Files\powersoft\pb6\it\examples\webpbsrv.exe".   The starting directory should be the same as the one where the program file is located, for example: "C:\Program Files\powersoft\pb6\it\examples"

In order for the server application to locate the PowerBuilder 6.0 virtual machine DLLs, you must also add an entry to the Registry to identify the application path. Use the Regedit program in Windows to add a new key under Hkey_Local_Machine\Software\Microsoft\Windows\CurrentVersion\App Paths. The key should be named Webpbsrv.Exe and its value should identify the location of the PowerBuilder VM, for example: C:\Program Files\Powersoft\shared.

Preparing your Web Server for PB Plug-ins

Before you can view the PB Plug-in examples, you must configure the web server by registering the appropriate content types (MIME types) with your Web server.
To Add the Plugins to WebSite's Content types (MIME types):

  1. Run the WebSite Server Admin program accessible via the Start Menu or the the WebSite Program Group
  2. Click on the Mapping tab.
  3. Click on the Content Types radio button.
  4. Type .pbd in the Field Extension (Class) entry field.
  5. Type application/vnd.powerbuilder6 in the MIME Content Type entry field.
  6. Click the Add button.
  7. Type .psr in the Field Extension (Class) entry field.
  8. Type application/datawindow in the MIME Content Type entry field.
  9. Click the Add button.
  10. Click the Ok button.
  11. Click Yes in the Immediate Reset message box.

Running the Examples

After all of the previous Installation Steps have been sucessfully completed!

  1. Run the Distributed PowerBuilder server application Webpbsrv.exe from the IT\Examples directory.
  2. Start the WebSite server from the Start Menu in the WebSite 1.1 program group by clicking on WebSite Server. You can tell the WebSite server is running when the WebSite icon appears in the notification area of the taskbar.
  3. Open your favorite browser and open the following URL: 
    4.          http://localhost/sample.htm

If you find that from a Web browser you are unable to run these samples, please review the installation steps on this page to ensure that all steps have been completed successfully.
